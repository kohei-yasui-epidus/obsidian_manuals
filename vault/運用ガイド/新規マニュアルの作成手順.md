# 新規マニュアルの作成手順(Obsidianで直接作る)

OneNoteを経由せず、Obsidian上で直接マニュアルを作成する手順です。

## 1. ノートを作成

1. 左のツリーで `マニュアル/<カテゴリ>` フォルダを右クリック →「新規ノート」
   (適切なカテゴリフォルダがなければ作成してOK)
2. ファイル名 = マニュアルのタイトル(例: `経費精算システムの操作手順`)
3. `Cmd + P` →「**テンプレート: テンプレートを挿入**」→「**マニュアルテンプレート**」を選択

## 2. frontmatter(ノート上部のプロパティ)を記入

| 項目 | 内容 |
| --- | --- |
| `category` | フォルダと同じカテゴリ名(例: `経理部`) |
| `status` | 作成中は `下書き` → 完成したら `公開` に変更 |
| `owner` | 担当者名(更新責任者) |

- `status: 下書き` のうちは [[マニュアル一覧]] の「下書き」欄に表示され、公開一覧には出ません
- 廃止するときは削除せず `status: アーカイブ` に変更します

## 3. 本文を書く

- 見出し(`##`)で手順を区切る。スクリーンショットはドラッグ&ドロップで貼り付け(自動で `添付ファイル/` に保存されます)
- 関連マニュアルには `[[ ]]` でリンクを張る(入力すると候補が出ます)

## 4. 公開と通達

1. `status` を `公開` に変更し、`updated` を今日の日付にする
2. **重要なマニュアルの場合は通達を出す**:
   `通達` フォルダに新規ノート → テンプレート「**通達テンプレート**」を挿入 → 記入
3. `Cmd + P` →「**Git: Commit-and-sync**」で共有

通達は [[Home]] の最上部に表示され、全員の目に入ります。

## 既存マニュアルの更新

1. 編集したら frontmatter の `updated` を今日の日付に変更(Homeの「🔄 最近更新」に載ります)
2. 大きな変更なら通達も作成
3. Git: Commit-and-sync で共有
