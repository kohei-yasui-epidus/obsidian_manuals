# チーム運用ルール(Git同期)

## 同期の仕組み

- Vault は Git リポジトリ(`vault/` サブフォルダ)として管理されています
- **Obsidian Git プラグイン**が 10分ごとに自動 pull し、常に最新版が手元に届きます
- 編集を共有するときは `Cmd + P` →「**Git: Commit-and-sync**」(pull → commit → push を一括実行)

> [!warning] デスクトップ版限定
> Obsidian Git の自動同期はデスクトップ版(Mac/Windows)で動作します。モバイル版での編集は非対応です(閲覧は他の同期手段があれば可)。

## 運用ルール

1. **編集する前に pull**(自動でも走りますが、編集直前は `Cmd + P` →「Git: Pull」が安全)
2. **こまめに Commit-and-sync**(溜めるほど競合しやすくなります)
3. **同じマニュアルを同時に編集しない** — `owner`(担当者)が更新責任を持ちます
4. マニュアルの**削除はしない** — `status: アーカイブ` に変更して残す
5. `.obsidian` の設定変更(プラグイン追加など)は管理者に相談してから

## 競合(コンフリクト)が起きたら

1. Obsidian Git が通知を出します(該当ファイルに `<<<<<<<` マークが入ります)
2. 該当ファイルを開き、残す内容を選んでマークの行を削除
3. 再度「Git: Commit-and-sync」

解決が難しければ、そのまま触らず管理者へ連絡してください。

## 権限の目安

| 役割 | できること |
| --- | --- |
| 閲覧メンバー | 閲覧・検索(push権限なしのread-onlyでも運用可) |
| 編集メンバー | マニュアル作成・更新・通達の発行 |
| 管理者 | OneNote変換、カテゴリ再編、プラグイン・設定変更 |
